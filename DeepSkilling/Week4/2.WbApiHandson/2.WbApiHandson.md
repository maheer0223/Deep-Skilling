# 2.WbApiHandson - Web API with Swagger and Postman Integration

## Objectives & Conceptual Overview

### 1. Swagger Integration in ASP.NET Core Web API

- **Swashbuckle.AspNetCore NuGet Package:**
  Provides automatic OpenAPI / Swagger specification generation and an interactive UI for testing Web API endpoints directly in the browser.
- **`AddSwaggerGen` Configuration:**
  Configured in `Program.cs` / `Startup.cs` to inject metadata into the generated OpenAPI specification:
  ```csharp
  builder.Services.AddSwaggerGen(c =>
  {
      c.SwaggerDoc("v1", new OpenApiInfo
      {
          Title = "Swagger Demo",
          Version = "v1",
          Description = "ASP.NET Core Web API with Swagger and Postman Demonstration",
          TermsOfService = new Uri("http://www.example.com/terms"),
          Contact = new OpenApiContact() { Name = "John Doe", Email = "john@xyzmail.com", Url = new Uri("http://www.example.com") },
          License = new OpenApiLicense() { Name = "License Terms", Url = new Uri("http://www.example.com") }
      });
  });
  ```
- **`UseSwagger` & `UseSwaggerUI` Middleware:**
  Exposes the JSON endpoint and renders the interactive HTML UI:
  ```csharp
  app.UseSwagger();
  app.UseSwaggerUI(c =>
  {
      c.SwaggerEndpoint("/swagger/v1/swagger.json", "Swagger Demo");
      c.RoutePrefix = "swagger";
  });
  ```
- **`[ProducesResponseType]` Attribute:**
  Specifies the HTTP status codes and response models returned by an action method, enhancing Swagger API documentation and client generation:
  ```csharp
  [ProducesResponseType(StatusCodes.Status200OK, Type = typeof(IEnumerable<Employee>))]
  [ProducesResponseType(StatusCodes.Status404NotFound)]
  ```

### 2. Postman Tool Structure & Usage

- **Postman Layout & Structure:**
  - **Left Sidebar:** Collections pane to organize, save, and group Web API requests into logical test suites.
  - **Center Pane Tabs:** Dedicated workspace tabs for individual HTTP requests.
  - **HTTP Method Dropdown:** Choice of action verbs (`GET`, `POST`, `PUT`, `DELETE`).
  - **URL Bar:** Target endpoint address (e.g., `http://localhost:5120/api/Emp`).
  - **Headers Tab:** Key-value pairs such as `Content-Type: application/json` or `Authorization: Bearer <token>`.
  - **Body Tab:** Input payload tab configured to `raw` -> `JSON` for `POST` and `PUT` requests.
  - **Response Pane:** Displays HTTP status code (`200 OK`, `201 Created`), time, size, and response body payload.

---

### 3. Route Attribute, Name Attribute & ActionName

- **Route Attribute Modification (`api/Emp`):**
  By default, `[Route("api/[controller]")]` resolves to `api/Employee`. Modifying the attribute to `[Route("api/Emp")]` explicitly updates the route template so clients access the service via `/api/Emp`.
- **`Name` Attribute in HTTP Verbs:**
  `[HttpGet(Name = "GetEmployeeById")]` assigns a user-friendly route name to an action method. This route name can be referenced using `CreatedAtRoute("GetEmployeeById", ...)` when returning `201 Created` responses.
- **`ActionName` Attribute:**
  `[ActionName("GetEmpList")]` allows developers to customize the internal action name or maintain multiple methods under identical action verbs without route collision.

## Code Implementation

### 1. `Models/Employee.cs`
```csharp
namespace WebAPIHandson.Models
{
    public class Employee
    {
        public int Id { get; set; }
        public string Name { get; set; } = string.Empty;
        public string Department { get; set; } = string.Empty;
        public decimal Salary { get; set; }
    }
}
```

### 2. `Controllers/EmployeeController.cs`
```csharp
using Microsoft.AspNetCore.Http;
using Microsoft.AspNetCore.Mvc;
using System.Collections.Generic;
using System.Linq;
using WebAPIHandson.Models;

namespace WebAPIHandson.Controllers
{
    [Route("api/Emp")]
    [ApiController]
    public class EmployeeController : ControllerBase
    {
        private static readonly List<Employee> _employees = new List<Employee>
        {
            new Employee { Id = 101, Name = "Alice Smith", Department = "IT", Salary = 75000 },
            new Employee { Id = 102, Name = "Bob Johnson", Department = "HR", Salary = 62000 },
            new Employee { Id = 103, Name = "Charlie Brown", Department = "Finance", Salary = 80000 }
        };

        [HttpGet(Name = "GetAllEmployees")]
        [ActionName("GetEmpList")]
        [ProducesResponseType(StatusCodes.Status200OK, Type = typeof(IEnumerable<Employee>))]
        public ActionResult<IEnumerable<Employee>> Get()
        {
            return Ok(_employees);
        }

        [HttpGet("{id}", Name = "GetEmployeeById")]
        [ProducesResponseType(StatusCodes.Status200OK, Type = typeof(Employee))]
        [ProducesResponseType(StatusCodes.Status404NotFound)]
        public ActionResult<Employee> Get(int id)
        {
            var emp = _employees.FirstOrDefault(e => e.Id == id);
            if (emp == null) return NotFound($"Employee with ID {id} not found.");
            return Ok(emp);
        }

        [HttpPost(Name = "CreateEmployee")]
        [ProducesResponseType(StatusCodes.Status201Created, Type = typeof(Employee))]
        public ActionResult Post([FromBody] Employee newEmp)
        {
            _employees.Add(newEmp);
            return CreatedAtRoute("GetEmployeeById", new { id = newEmp.Id }, newEmp);
        }
    }
}
```
