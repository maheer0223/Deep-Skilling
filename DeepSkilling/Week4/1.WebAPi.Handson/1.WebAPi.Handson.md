# 1.WebAPi.Handson - ASP.NET Core Web API Guided Hands-On

## Objectives & Theory Overview

### 1. RESTful Web Service, Web API & Microservices

- **Representational State Transfer (REST):** An architectural style for designing networked applications. REST uses standard HTTP protocols and operates on resources represented by URIs (e.g., `/api/values`).
- **Key Features of REST Architecture:**
  - **Representational State Transfer:** Resources are decoupled from their representations. Data can be represented as JSON, XML, Plain Text, etc.
  - **Statelessness:** Every HTTP request from a client contains all the information needed by the server to fulfill that request. No client context is stored on the server between requests.
  - **Messages:** Communication takes place via standard HTTP request and response messages with headers, body, and status codes.
  - **Not Restricted to XML:** Unlike traditional SOAP services which mandate XML, RESTful Web APIs support JSON, XML, HTML, and binary payloads. JSON is the modern standard due to its lightweight nature.
- **Concept of Microservices:**
  - An architectural pattern where an application is decomposed into small, independent, loosely-coupled services that communicate over lightweight protocols (typically HTTP REST Web APIs).
- **Difference between Web Service & Web API:**
  | Feature | Web Service (SOAP / WCF) | Web API (RESTful) |
  | :--- | :--- | :--- |
  | **Protocol** | Primarily HTTP/HTTPS, SMTP, TCP via SOAP | Standard HTTP/HTTPS |
  | **Format** | Strictly XML | JSON, XML, Form Data, Plain Text |
  | **Architecture** | Heavyweight, contract-based (WSDL) | Lightweight, Resource-based (URI) |
  | **Clients** | Compatible with specific SOAP clients | Any client (Browsers, Mobile, IoT, Postman) |

---

### 2. HttpRequest & HttpResponse

- **HttpRequest:** Sent by the client to the server. Contains:
  - HTTP Method (GET, POST, PUT, DELETE)
  - Request URI (`/api/values/1`)
  - Request Headers (`Accept`, `Content-Type`, `Authorization`)
  - Request Body (JSON payload for POST/PUT)
- **HttpResponse:** Returned by the server to the client. Contains:
  - Status Code & Reason Phrase (`200 OK`, `400 Bad Request`, `404 Not Found`)
  - Response Headers (`Content-Type`, `Date`, `Server`)
  - Response Body (Serialized data object, e.g., JSON)

---

### 3. Action Verbs & Attributes

| Action Verb | Meaning | Attribute Declaration | Idempotent |
| :--- | :--- | :--- | :--- |
| **HttpGet** | Retrieves data from the server | `[HttpGet]`, `[HttpGet("{id}")]` | Yes |
| **HttpPost** | Creates a new resource on the server | `[HttpPost]` | No |
| **HttpPut** | Updates an existing resource (or replaces it) | `[HttpPut("{id}")]` | Yes |
| **HttpDelete** | Removes a resource from the server | `[HttpDelete("{id}")]` | Yes |

---

### 4. HTTP Status Codes & Action Result Types

- **`Ok()` (HTTP 200 / 201):** Returned when request succeeds (`200 OK` or `201 Created`).
- **`BadRequest()` (HTTP 400):** Returned when payload or parameters are invalid.
- **`Unauthorized()` (HTTP 401):** Returned when authentication is required or fails.
- **`NotFound()` (HTTP 404):** Returned when requested resource ID does not exist.
- **`StatusCode(500)` (HTTP 500):** Returned when an unhandled server error occurs.

---

### 5. Web API Controller Structure (.NET Core vs .NET 4.5 Framework)

- **Controller Inheritance:**
  - `.NET Framework 4.5`: Inherits from `ApiController` (`System.Web.Http.ApiController`).
  - `.NET Core / .NET 8+`: Inherits from `ControllerBase` (`Microsoft.AspNetCore.Mvc.ControllerBase`) paired with `[ApiController]` attribute.
- **Attributes:**
  - `[ApiController]`: Enables API-specific behaviors like automatic model validation, HTTP 400 response generation, and binding source parameter inference (`[FromBody]`).
  - `[Route("api/[controller]")]`: Defines URI path based on controller name.

---

### 6. Configuration Files in Web API

1. **`Program.cs` / `Startup.cs`:**
   - Registers services into the Dependency Injection (DI) container (`builder.Services.AddControllers()`).
   - Configures middleware pipeline (`app.UseAuthorization()`, `app.MapControllers()`).
2. **`appsettings.json` / `appsettings.Development.json`:**
   - Stores application configuration settings (Connection strings, Logging levels, External API keys).
3. **`launchSettings.json`:**
   - Defines local launch profiles (IIS Express, Kestrel), environment variables (`ASPNETCORE_ENVIRONMENT`), application URLs, and SSL ports.
4. **`Route.config` & `WebApiConfig.cs` (.NET 4.5 Framework):**
   - `Route.config`: Configured MVC convention routes (`{controller}/{action}/{id}`).
   - `WebApiConfig.cs`: Configured REST Web API routes (`api/{controller}/{id}`) and JSON formatters in legacy .NET Framework. In ASP.NET Core, routing is consolidated into `Program.cs` with attribute routing (`[Route]`).

## Code Implementation: `ValuesController.cs`

```csharp
using Microsoft.AspNetCore.Mvc;
using System.Collections.Generic;

namespace WebAPIHandson.Controllers
{
    [Route("api/[controller]")]
    [ApiController]
    public class ValuesController : ControllerBase
    {
        private static readonly List<string> _values = new List<string> { "value1", "value2", "value3" };

        // GET: api/values
        [HttpGet]
        public ActionResult<IEnumerable<string>> Get()
        {
            return Ok(_values);
        }

        // GET api/values/5
        [HttpGet("{id}")]
        public ActionResult<string> Get(int id)
        {
            if (id < 0) return BadRequest("ID must be positive.");
            if (id >= _values.Count) return NotFound($"Value with ID {id} not found.");
            return Ok(_values[id]);
        }

        // POST api/values
        [HttpPost]
        public ActionResult Post([FromBody] string value)
        {
            if (string.IsNullOrWhiteSpace(value)) return BadRequest("Value cannot be empty.");
            _values.Add(value);
            int newId = _values.Count - 1;
            return CreatedAtAction(nameof(Get), new { id = newId }, new { id = newId, value });
        }

        // PUT api/values/5
        [HttpPut("{id}")]
        public ActionResult Put(int id, [FromBody] string value)
        {
            if (id < 0 || id >= _values.Count) return NotFound($"Value with ID {id} not found.");
            _values[id] = value;
            return Ok(new { message = $"Value at ID {id} updated.", updatedValue = value });
        }

        // DELETE api/values/5
        [HttpDelete("{id}")]
        public ActionResult Delete(int id)
        {
            if (id < 0 || id >= _values.Count) return NotFound($"Value with ID {id} not found.");
            string removedValue = _values[id];
            _values.RemoveAt(id);
            return Ok(new { message = $"Value at ID {id} deleted.", deletedValue = removedValue });
        }
    }
}
```
